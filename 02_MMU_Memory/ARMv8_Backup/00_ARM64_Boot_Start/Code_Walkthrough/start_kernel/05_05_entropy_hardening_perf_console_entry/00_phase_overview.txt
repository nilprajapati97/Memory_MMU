Phase: 05
Primary purpose: Entropy completion, hardening gates, perf hooks, and first robust console-driven diagnosability

From-scratch explanation:
- This phase groups calls that must be completed together before the next boot capability level is safe.
- Think of this as a contract boundary: once complete, the kernel is allowed to assume additional capabilities.

Detailed goals:

2) Avoid exposing partially initialized global state to asynchronous paths.
3) Prepare predictable failure diagnostics if bring-up fails in this region.
