# 00_My_Scratch

A personal scratch repository containing study notes, experiments, reference material, and proof-of-concept code covering systems programming, the Linux kernel, embedded systems, RTOS, ARM architecture, and interview/company-specific preparation.

## Repository Structure

| Directory | Description |
|-----------|-------------|
| [`00_Companies/`](./00_Companies) | Company-specific interview preparation and practice material (NVIDIA, Cisco, Qualcomm, Google, etc.). |
| [`01_ReferenceDocs/`](./01_ReferenceDocs) | Reference documentation, including notes on *Understanding the Linux Kernel*. |
| [`02_MMU_Memory/`](./02_MMU_Memory) | Memory management notes — ARM64/ARMv8 references, MMU concepts, memory APIs/flows, and related Q&A. |
| [`03_Kernel_Experiment/`](./03_Kernel_Experiment) | Hands-on Linux kernel experiments: interrupts, system calls, drivers, and resource management. |
| [`04_RTOS_CPU/`](./04_RTOS_CPU) | Embedded C and RTOS notes and exercises. |
| [`05_Linux_System_Design/`](./05_Linux_System_Design) | Linux system design topics: single-threaded CPU design, schedulers, high-frequency ISR design guide, and questions. |
| [`06_Kernel_Devlopment/`](./06_Kernel_Devlopment) | Kernel development documentation. |
| [`07_My_Poc/`](./07_My_Poc) | Personal proof-of-concept work, including pre/post-silicon experiments. |

## Purpose

This repository serves as a personal knowledge base for:

- Learning and revisiting **Linux kernel internals**
- Studying **ARMv8 / ARM64** architecture and memory management
- Practicing **embedded C** and **RTOS** concepts
- Exploring **Linux system design** patterns
- Preparing for **systems-engineering interviews**

## Usage

Each top-level directory is self-contained. Browse into any folder to find topic-specific notes, code, or reference material. Many subdirectories include their own `ReadMe` files with further detail.
